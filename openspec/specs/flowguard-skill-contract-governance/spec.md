# flowguard-skill-contract-governance Specification

## Purpose

Define the required entrypoint, prompt metadata, generated contract, native-check ownership, and deep-validation rules shared by all seventeen FlowGuard skills while preserving each route's distinct activation and claim boundary.

## Requirements
### Requirement: Standard Skill Entrypoint Contract
Every FlowGuard skill SHALL contain the exact entrypoint sections `Purpose`, `Entrypoint Scope`, `Local Material Routing`, `Entrypoint Acceptance Map`, `Use When`, `Do Not Use When`, `Required Workflow`, `Hard Gates`, `Output Requirements`, and `SkillGuard Maintenance`. The sections MUST state route-specific behavior rather than an interchangeable generic template.

#### Scenario: Required section is absent
- **WHEN** any suite member omits one required entrypoint section
- **THEN** static skill validation fails for that member

#### Scenario: Unrelated skills share a generic contract
- **WHEN** two unrelated skills expose identical route ids, workflow stages, obligations, and checks
- **THEN** depth validation fails with a generic-contract or parallel-route-risk finding

### Requirement: Prompt Metadata Parity
Every suite member SHALL provide `agents/openai.yaml`. Its default prompt MUST declare precise trigger, route role, primary evidence gate, non-owned work, required output fields, and claim boundary. Stable route id and role MUST agree across YAML, `SKILL.md`, contract source, and the route registry.

#### Scenario: YAML route role drifts
- **WHEN** `openai.yaml` calls a delegated mode a public owner while the registry classifies it as delegated
- **THEN** prompt-parity validation fails and identifies each conflicting source

### Requirement: Target-Specific Contract Source
Every skill SHALL own `.skillguard/contract-source.json` containing its source requirements, native route ids, phase bindings, workflow stages, native check bindings, acceptance obligations, skill-specific checks, test-gap plan, closure blockers, layout profiles, and non-parallel-route proof.

#### Scenario: Behavior ledger lacks a control root
- **WHEN** the Behavior Commitment Ledger has no contract source or required control files
- **THEN** suite certification fails as a missing-control-root blocker

#### Scenario: Native check binding is absent
- **WHEN** a contract source declares an acceptance obligation without a native check or explicit test-gap disposition
- **THEN** contract compilation fails and no work contract is certified

### Requirement: Deterministic Deep Contract Generation
The contract compiler SHALL deterministically generate the current SkillGuard `work-contract.json`, `check_manifest.json`, and contract hash from the skill prompt, contract source, route registry, and compiler schema. Check mode MUST fail when tracked generated files are stale or manually altered.

#### Scenario: Skill prompt changes without regeneration
- **WHEN** `SKILL.md` changes and the generated contract hash still reflects the prior input
- **THEN** contract check fails with a stale-generated-contract finding

#### Scenario: Compiler runs twice
- **WHEN** unchanged inputs are compiled twice
- **THEN** the generated bytes and contract hash are identical

### Requirement: Native Integrated Ownership
Every FlowGuard SkillGuard contract SHALL declare `native-integrated`, `run_record_required=false`, and evidence that SkillGuard wraps the FlowGuard-owned route/check surface without creating a duplicate execution owner.

#### Scenario: Contract declares a parallel execution path
- **WHEN** a contract assigns SkillGuard ownership of a native FlowGuard workflow stage
- **THEN** depth validation fails with a parallel-route-risk finding

### Requirement: Seventeen Skill Deep Certification
Full skill contract governance SHALL require static skill, contract, and depth validation to pass for all seventeen canonical suite members with zero hollow-contract, parallel-route-risk, legacy-schema, missing-control, or stale-generation findings.

#### Scenario: Sixteen members pass and one is hollow
- **WHEN** sixteen members pass but one member lacks required deep evidence
- **THEN** suite certification fails and reports `16/17`, not a partial suite pass
