## ADDED Requirements

### Requirement: SkillGuard supervises only FlowGuard-declared purpose checks
FlowGuard SHALL remain the sole owner of model-purpose semantics, protected-failure selection, fixtures, native oracles, and claim boundaries. SkillGuard SHALL only freeze the exact FlowGuard-declared check inventory, reconcile current owner receipts, enforce installation projection, and consume terminal closure. SkillGuard SHALL NOT classify Guard families, invent purpose contracts, or impose FlowGuard's good/bad proof pattern on unrelated skills.

#### Scenario: FlowGuard declares three native checks
- **WHEN** the current FlowGuard target declares three purpose-related native checks
- **THEN** SkillGuard requires exactly those three current owner receipts without adding its own domain check

#### Scenario: Ordinary skill declares one unrelated check
- **WHEN** a non-Guard skill declares one native check with no model-purpose semantics
- **THEN** SkillGuard supervises that one check and does not require a protected failure or good/bad pair

### Requirement: Installed FlowGuard projection consumes current purpose contracts
FlowGuard installation SHALL consume only the frozen installation projection after source contracts and affected native checks are current. Installed-currentness verification SHALL be read-only and SHALL NOT launch another validation owner.

#### Scenario: Source skill changed but installed projection is old
- **WHEN** a FlowGuard skill's purpose-loop contract changed and the installed projection does not match
- **THEN** installed behavior remains unsynchronized and cannot support the final claim
